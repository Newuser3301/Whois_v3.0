image:https://img.shields.io/badge/Python-3.11%2B-blue[Python,link=] image:https://img.shields.io/badge/GUI-PyQt6-green[PyQt6,link=] image:https://img.shields.io/badge/License-Educational-orange[License,link=] image:https://img.shields.io/badge/Platform-Windows%20%7C%20Linux%20%7C%20macOS-lightgrey[Platform,link=]

A powerful, modern GUI application built with Python and PyQt6 that streamlines the reconnaissance phase of penetration testing. Consolidates *WHOIS*, *DNS enumeration*, *passive OSINT*, *WAF/CDN fingerprinting*, *Nmap port scanning*, *web technology analysis*, *CVE detection*, and *TLS certificate analysis* into a single, intuitive interface.

Results are displayed in organized tabs and can be exported to *JSON* or *TXT* reports with one click.

IMPORTANT: Use this tool *only* on domains you own or have explicit written permission to test. Nmap port scanning constitutes active reconnaissance and may be illegal without proper authorization.

---

== ✨ Key Features

[cols="1,3", options="header"]
|===
|Tab
|Capability

|*WHOIS*
|Retrieves registrar, registration dates, domain status, DNSSEC, abuse contacts, and name servers.

|*DNS Records*
|Queries A, AAAA, MX, TXT, NS, CNAME, SOA, CAA records with individual or bulk refresh.

|*Subdomains*
|Discovers subdomains via `subfinder` (if installed) or 10+ public passive sources.

|*OSINT*
|Harvests emails, URLs, interesting paths, passive hosts — 100% passive reconnaissance.

|*WAF / Firewall*
|Fingerprints WAF/CDN providers via DNS markers, IP ranges, HTTP headers, cookies, and server signatures.

|*Nmap Ports*
|Scans top 100 TCP ports using `python-nmap` — displays only open ports.

|*Tech & CVE*
|Detects web technologies via BuiltWith and HTTP headers, then searches matching CVEs via `nvdlib`.

|*TLS Analysis*
|Deep-dives into TLS certificates: SAN, issuer, validity, ciphers, protocols, and anomaly indicators.

|*Attack Vectors*
|Summarizes risky findings: DNSSEC gaps, SPF/DMARC misconfigurations, exposed origin IPs, and more.

|*Raw Output*
|Full raw JSON/text view of all collected data for debugging and custom parsing.

|*Report*
|Export results to JSON or TXT, or copy everything to clipboard instantly.
|===

== 📋 Requirements

=== Python Version

*Python 3.11+* is recommended. This tool has been tested and runs smoothly on Python 3.15.

=== Python Libraries

Install all required dependencies:

[source,powershell]
----
pip install PyQt6 python-whois dnspython python-nmap requests cryptography pyOpenSSL builtwith nvdlib Wappalyzer
----

NOTE: The `Wappalyzer` package may install Playwright dependencies. The current implementation primarily uses `builtwith` and HTTP header fingerprints; Wappalyzer is available for future expansion.

=== External Tools

*Nmap* must be installed and accessible via PATH for `python-nmap` to function.

Windows installation:

[source,text]
----
https://nmap.org/download.html
----

Verify installation:

[source,powershell]
----
Get-Command nmap
----

For enhanced subdomain discovery (optional):

[source,powershell]
----
subfinder -version
----

If `subfinder` is not installed, the tool gracefully falls back to public passive sources.

Navigate to the project directory and run:

----

. Enter a domain (e.g., `example.com`)
. Click the *RECON START* button
. Browse results across all tabs

== ⚡ Performance & Asynchronous Design

The recon engine leverages `asyncio` and `ThreadPoolExecutor` to execute independent tasks in parallel:

- WHOIS lookup
- DNS queries
- Passive subdomain enumeration
- TLS certificate analysis
- Technology detection
- Nmap port scanning
- WAF/CDN fingerprinting
- OSINT URL/email harvesting
- CVE search

If a single module fails, the entire scan continues uninterrupted. The affected tab shows a brief error status or message.

== 🌐 Passive Subdomain Sources

The following public sources are queried:

- `subfinder` (if installed locally)
- `crt.sh`
- CertSpotter
- AlienVault OTX
- HackerTarget
- URLScan
- RapidDNS
- Wayback CDX
- BufferOver
- ThreatMiner
- AnubisDB

Public sources may occasionally return `429`, `500`, `503`, DNS errors, or timeouts. This is expected behavior. The GUI reflects these statuses in the *OSINT → Sources* panel as `rate limited`, `timeout`, or `service unavailable`.

== 🔍 Nmap Port Scanning

The Nmap tab executes with the following arguments:

[source,text]
----
-Pn -T3 --top-ports 100 --open
----

This scans only the top 100 TCP ports and displays exclusively open ones.

CAUTION: Nmap is an active scan. Use *only* on authorized targets.

== 🔬 Technology & CVE Detection

Technology detection methods:

- `builtwith.parse()`
- HTTP header fingerprinting

CVE lookup:

- `nvdlib.searchCVE()`
- Limited CVE queries per detected technology

NOTE: The NVD API may enforce rate limits. When this occurs, status indicators in the *TECH/CVE → Sources* or *CVE Matches* panel reflect the condition.

== 🛠️ Troubleshooting

=== GUI Fails to Launch

Verify PyQt6 installation:

[source,powershell]
----
python -c "import PyQt6; print('ok')"
----

=== Nmap Not Working

Check both the Python library and system executable:

[source,powershell]
----
python -c "import nmap; print('python-nmap ok')"
Get-Command nmap
----

If `Get-Command nmap` returns nothing, install Nmap and add it to PATH.

=== Passive Source Errors

`429`, `500`, `503`, `timeout`, and DNS errors are common with free public OSINT services. The tool gracefully continues with remaining sources.

=== No CVE Results

Possible reasons:

- NVD API temporarily unavailable
- Rate limiting in effect
- Technology name is too generic
- Version information not detected

=== TLS Errors

Possible reasons:

- Port 443 is closed
- SNI not supported by the server
- Certificate chain issues
- Network or proxy errors

== ⚠️ Current Limitations

- WAF fingerprinting is heuristic and may produce false positives/negatives
- CVE searches based on unversioned technology names may yield false positives
- Passive OSINT sources are inherently unstable; rate limits and timeouts are expected
- Subdomain tab displays only the first 1000 results for GUI performance; full list is preserved in RAW/JSON exports
- Nmap scans are limited to the top 100 TCP ports
